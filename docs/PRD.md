# PRD: 每日待辦事項 (Daily To-Do List)

## 專案概述
- **專案名稱**：每日待辦事項 (Daily To-Do List)
- **專案描述**：這是一個基於 Web 的待辦事項管理系統，幫助使用者記錄、追蹤並管理每天的任務，提升工作與生活效率。
- **背景與動機**：現代人生活忙碌，任務繁多，需要一個簡單直觀的工具來記錄每天的待辦事項，避免遺漏重要工作並減輕記憶負擔。
- **目標用戶**：學生、上班族、自由工作者，以及任何需要管理日常任務的人。
- **核心價值主張**：簡單、直觀、快速上手，提供極佳的視覺體驗與流暢的操作，讓任務管理不再是負擔。

## 功能需求
1. **新增任務**：作為使用者，我希望能夠新增一個任務，以便記錄需要完成的事情。
2. **檢視任務列表**：作為使用者，我希望能夠看到所有未完成和已完成的任務，以便隨時了解目前的進度。
3. **標記完成/未完成**：作為使用者，我希望能夠點擊將任務標記為完成或取消完成，以便追蹤任務狀態。
4. **刪除任務**：作為使用者，我希望能夠刪除不再需要的任務，以便保持列表的整潔。
5. **編輯任務**：作為使用者，我希望能夠修改已經建立的任務內容，以便更正錯誤或更新資訊。

## 非功能需求
- **技術限制**：
  - 後端：Python Flask
  - 前端：Jinja2 模板 + 現代化 Vanilla HTML/CSS/JS (將套用精緻的 UI 設計，例如流暢動畫與現代化色彩學)
  - 資料庫：SQLite
- **效能與安全考量**：
  - 防範常見的網頁攻擊（例如 XSS 與 SQL Injection，透過 Flask 與 ORM/參數化查詢預防）。
  - 確保網頁在不同裝置（手機、平板、電腦）上都能夠響應式(RWD)正常顯示。

## MVP 範圍 (Minimum Viable Product)
- **Must Have (必須有)**：
  - 新增任務
  - 檢視任務列表
  - 標記完成/未完成
  - 刪除任務
- **Should Have (應該有)**：
  - 編輯任務內容
- **Nice to Have (可以有)**：
  - 標籤分類功能
  - 深色模式 (Dark Mode)
  - 拖曳排序

## 專案成員與分工
| 角色 | 姓名/負責人 | 負責項目 |
|---|---|---|
| 組員 | User | 任務內容編輯與更新（點擊任務後，可修改其文字內容與編輯） |
| AI 開發助理 | Antigravity | 協助「任務編輯」功能的系統設計、前後端程式實作與 UI 設計 |
