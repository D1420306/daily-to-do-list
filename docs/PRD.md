# 產品需求文件 (PRD)

## 1. 專案概述
- **專案名稱**：每日代辦事項 (Daily To-Do List)
- **專案描述**：這是一個簡單的每日任務管理系統，讓使用者可以管理日常任務，特別著重於任務狀態的快速切換（待辦/已完成）。
- **背景與動機**：現代人生活忙碌，需要一個簡單明瞭的工具來記錄每天必須完成的事項，並且能夠直覺地標記完成狀態以獲得成就感與掌握進度。
- **目標用戶**：需要輕量級任務管理工具的一般大眾。
- **核心價值主張**：提供極簡且直覺的介面，快速紀錄任務，並透過點擊勾選框即可一鍵切換狀態。

## 2. 功能需求
以下為本專案的主要功能清單：

- **使用者故事 (User Stories)**：
  1. **查看任務**：作為使用者，我希望「能看到所有代辦與已完成的任務列表」，以便「掌握今天的進度」。
  2. **新增任務**：作為使用者，我希望「能輸入文字並新增任務」，以便「記錄新的待辦事項」。
  3. **切換狀態**：作為使用者，我希望「點擊任務旁邊的勾選框可將其標記為已完成，再次點擊則恢復為待辦」，以便「快速更新任務的執行狀態」。
  4. **編輯任務**：作為使用者，我希望「能修改既有任務的內容」，以便「在輸入錯誤或需求改變時進行更正」。
  5. **刪除任務**：作為使用者，我希望「能刪除不需要的任務」，以便「保持清單的整潔」。

## 3. 非功能需求
- **技術限制**：
  - 後端架構：Python + Flask
  - 頁面渲染：Jinja2 (與 Flask 整合，不需要前後端分離)
  - 資料庫：SQLite (透過 sqlite3 原生操作或 SQLAlchemy)
- **效能與安全考量**：
  - 防止 SQL Injection（使用參數化查詢）。
  - 基本的表單驗證（防止空任務提交）。
  - 前端使用 Bootstrap 5 提供良好的響應式體驗 (RWD) 與狀態切換視覺反饋。

## 4. MVP 範圍
- **Must Have (必須具備)**：
  - 任務列表顯示。
  - 新增任務功能。
  - 任務狀態切換（點選勾選框切換「待辦」與「已完成」）。
  - 刪除任務。
- **Should Have (應該具備)**：
  - 編輯任務文字。
- **Nice to Have (加分項目)**：
  - 動畫效果（例如標記完成時的刪除線動畫）。
