# PRD: 每日代辦 (Daily To-Do List)

## 1. 專案概述

### 背景與動機
現代人生活忙碌，常常忘記每日該處理的瑣事。需要一個簡單直覺的工具來記錄每日任務。

### 目標用戶
需要管理個人日常任務的學生、上班族或自由工作者。

### 核心價值主張
提供快速、無需複雜學習曲線的任務管理體驗，幫助使用者專注於完成當下的代辦事項。

## 2. 功能需求

1. **任務新增**：作為使用者，我希望可以快速輸入任務名稱並新增，以便記錄我想做的事。
2. **任務列表查看**：作為使用者，我希望可以一覽所有未完成與已完成的任務，以便掌握進度。
3. **任務狀態切換**：作為使用者，我希望可以勾選完成或取消完成任務，以便追蹤執行狀態。
4. **任務刪除與清理**：作為使用者，我希望可以針對單一任務進行刪除，或一鍵清理已完成的項目，以便保持列表整潔。
5. **任務編輯**：作為使用者，我希望可以修改現有任務的文字內容，以便更正錯誤或更新細節。

## 3. 非功能需求

### 技術限制
- 後端：Flask 框架
- 前端視圖：Jinja2 模板渲染
- 資料庫：SQLite

### 效能與安全考量
- 確保頁面載入迅速，避免過多的前端資源。
- 防止常見的 Web 攻擊（如 XSS），並對使用者輸入進行驗證與跳脫。

## 4. MVP 範圍

### Must Have
- 任務的 CRUD (新增、讀取、更新、刪除)。
- 任務完成狀態的切換。
- 一鍵清理已完成的任務。

### Should Have
- 簡單直覺的 UI 設計（包含深色模式、現代化設計美學）。
- 區分「未完成」與「已完成」的任務區塊。

### Nice to Have
- 任務分類標籤 (Tags)。
- 截止時間設定。
