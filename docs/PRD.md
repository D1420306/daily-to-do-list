# PRD - 每日代辦事項 (Daily To-Do List)

## 1. 專案概述
- **背景與動機**：現代人生活忙碌，常常會忘記每天需要完成的瑣事。本專案旨在提供一個簡單、直覺的網頁介面，讓使用者能快速記錄每天的代辦事項。
- **目標用戶**：需要輕量級任務記錄工具的學生、上班族或一般大眾。
- **核心價值主張**：以最少的點擊次數完成任務的記錄，介面乾淨、操作直覺。

## 2. 功能需求

以下為系統的主要功能：

1. **新增任務**
   - 作為 **使用者**，我希望 **可以在輸入框輸入文字並送出**，以便 **將新的代辦事項加入清單中**。
2. **檢視任務清單**
   - 作為 **使用者**，我希望 **可以在首頁看到所有已建立的代辦事項**，以便 **知道今天還有哪些事情要完成**。

## 3. 非功能需求

- **技術限制**：
  - 後端：Python 搭配 Flask 框架
  - 前端：HTML + Vanilla CSS + Jinja2 模板引擎
  - 資料庫：SQLite 
- **效能與安全考量**：
  - 確保防範基本的 XSS 攻擊（Jinja2 預設會逸出渲染的文字）。
  - 網頁須具備基礎的響應式設計（RWD）。

## 4. MVP 範圍 (最小可行性產品)

- **Must Have (必須有)**
  - 輸入文字並新增任務。
  - 在頁面上顯示所有已建立的任務。
  - 基礎的視覺設計，確保乾淨好閱讀。
- **Nice to Have (加分項目)**
  - 漂亮的深色模式（Dark Mode）。
