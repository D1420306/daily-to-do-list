# 每日代辦事項 (Daily To-Do List) PRD

## 1. 專案概述
- **專案名稱**：每日代辦事項 (Daily To-Do List)
- **專案描述**：一個幫助使用者記錄、追蹤並管理每日待辦任務的網站，旨在提升個人的工作與生活效率，減少遺忘重要事項的風險。
- **背景與動機**：現代人生活忙碌，常常會忘記瑣碎但重要的事情。市面上雖然有許多待辦事項軟體，但我們需要一個簡單、直覺、且能自己掌控的輕量級網頁應用程式。
- **目標用戶**：需要管理日常任務的學生、上班族，或任何想提升效率的人。
- **核心價值主張**：簡單易用、一目瞭然，幫助使用者專注於「今天」要完成的事。

## 2. 功能需求
我們將提供以下主要功能：

1. **任務新增 (Add Task)**
   - 作為 **使用者**，我希望 **能夠輸入任務標題並新增任務**，以便 **記錄我需要做的事情**。
2. **任務優先級標籤 (Priority Tags) - 核心功能**
   - 作為 **使用者**，我希望 **在新增或編輯任務時，能為任務加上「高」、「中」、「低」三種顏色標籤**，以便 **區分任務的重要性與急迫性，幫助我決定執行順序**。
3. **任務列表查看 (View Tasks)**
   - 作為 **使用者**，我希望 **能看到所有待辦與已完成的任務清單，並清楚看到每個任務的優先級顏色**，以便 **掌握目前的進度與重點**。
4. **任務狀態切換 (Toggle Task Status)**
   - 作為 **使用者**，我希望 **能勾選任務將其標記為完成（或取消完成）**，以便 **清楚知道哪些事情已經做完**。
5. **任務刪除 (Delete Task)**
   - 作為 **使用者**，我希望 **能刪除不需要或錯誤新增的任務**，以便 **保持清單的整潔**。
6. **任務編輯 (Edit Task)**
   - 作為 **使用者**，我希望 **能修改既有任務的內容與優先級標籤**，以便 **在計畫改變時更新任務資訊**。

## 3. 非功能需求
- **技術限制**：
  - 後端：Python + Flask
  - 模板引擎：Jinja2
  - 資料庫：SQLite
  - 前端：HTML / CSS (Vanilla CSS, 打造充滿現代感、玻璃擬物化 (Glassmorphism) 或精美漸層的視覺設計) / Vanilla JavaScript
- **效能與安全考量**：
  - 確保基本的防止 XSS 攻擊 (透過 Jinja2 預設的跳脫機制)。
  - 確保資料庫寫入時防止 SQL Injection (透過 SQLAlchemy 或參數化查詢)。
  - 頁面載入速度需快速且互動流暢，並加入微動畫 (micro-animations) 提升體驗。

## 4. MVP 範圍 (Minimum Viable Product)
- **Must Have (必須要有)**:
  - 任務新增
  - **任務優先級標籤（高/中/低 三色標籤，您的主要負責項目）**
  - 任務列表查看
  - 任務狀態切換 (完成/未完成)
  - 任務刪除
- **Should Have (最好要有)**:
  - 任務編輯
  - 美觀且具現代感的 UI 設計 (漸層、動畫等)
- **Nice to Have (有也很好)**:
  - 依優先級排序任務
  - 期限設定與逾期提醒
  - 深色模式 (Dark Mode) 切換
