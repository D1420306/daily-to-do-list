# ROUTES DESIGN: 每日代辦 (Daily To-Do List) 路由與頁面設計

本文件根據 PRD 與流程圖，定義 Flask 的 URL 路由、對應的方法與處理邏輯。由於這是一個單頁面應用體驗的 Web App，所有視圖渲染都集中在單一路由上，其他路由則作為資料操作的端點（Endpoint），並在完成後 Redirect 回首頁。

## 1. 路由總覽表格

| 功能 | HTTP 方法 | URL 路徑 | 對應模板 | 說明 |
| --- | --- | --- | --- | --- |
| 任務首頁 | GET | `/` | `index.html` | 讀取所有任務並分為「未完成」與「已完成」區塊顯示 |
| 新增任務 | POST | `/add` | — | 接收表單 `title`，存入 DB，重導向至首頁 |
| 切換狀態 | POST | `/toggle/<int:task_id>` | — | 切換特定任務的 `is_completed` 狀態，重導向至首頁 |
| 刪除任務 | POST | `/delete/<int:task_id>` | — | 刪除特定任務，重導向至首頁 |
| 編輯任務 | POST | `/edit/<int:task_id>` | — | 接收表單 `title`，更新特定任務內容，重導向至首頁 |
| 清理已完成 | POST | `/clear-completed` | — | 刪除所有 `is_completed=True` 的任務，重導向至首頁 |

## 2. 每個路由的詳細說明

### 任務首頁
- **輸入**：無
- **處理邏輯**：查詢 `Task` Model，取得所有資料。可考慮在資料庫查詢時依據 `created_at` 排序。
- **輸出**：渲染 `index.html`，並傳入 `tasks` 變數。

### 新增任務
- **輸入**：表單欄位 `title`
- **處理邏輯**：驗證 `title` 是否空白。若不空白，建立 `Task` 物件並寫入資料庫。
- **輸出**：重導向至 `/`。
- **錯誤處理**：如果輸入為空，可透過 Flask `flash` 顯示錯誤訊息，然後重導向至 `/`。

### 切換狀態
- **輸入**：URL 參數 `task_id`
- **處理邏輯**：從資料庫查詢 `id` 為 `task_id` 的 `Task` 物件。將其 `is_completed` 值反轉 (`True` 變 `False`，`False` 變 `True`)，儲存並提交。
- **輸出**：重導向至 `/`。
- **錯誤處理**：若找不到對應的 task_id，回傳 404 或直接重導向至 `/`。

### 刪除任務
- **輸入**：URL 參數 `task_id`
- **處理邏輯**：從資料庫查詢 `id` 為 `task_id` 的 `Task` 物件，並刪除之。
- **輸出**：重導向至 `/`。

### 編輯任務
- **輸入**：URL 參數 `task_id`，表單欄位 `title`
- **處理邏輯**：從資料庫查詢對應任務，並更新其 `title` 欄位。
- **輸出**：重導向至 `/`。

### 清理已完成任務
- **輸入**：無
- **處理邏輯**：從資料庫刪除所有 `is_completed == True` 的 `Task` 物件。
- **輸出**：重導向至 `/`。

## 3. Jinja2 模板清單

- `base.html`：基底模板，包含共用的 `<head>`、CSS 引入、頁首、頁尾結構。
- `index.html`：繼承自 `base.html`，主要的任務操作介面。包含新增表單、未完成任務清單、已完成任務清單、以及任務項目的獨立表單。
